# data-science-ml-router

Use this router when:
- the user wants machine learning, model evaluation, forecasting, feature analysis, or tabular data science help
- the task sits between scientific computing and AI engineering rather than purely software or purely wet-lab bioinformatics

Routing guidance:
- Use AI-Research-SKILLs for modern AI research and LLM workflows.
- Use concrete libraries such as scikit-learn, pytorch-lightning, transformers, polars, statsmodels, or shap once the stack is clear.
- Use ai-learning-agentic-coding-and-prompts, ai-learning-rag-mcp-and-agent-systems, ai-learning-ai-for-biology-and-research, ai-learning-project-architecture-and-iteration, ai-learning-research-productivity-and-editing, bio-learning-survival-modeling-and-structured-prediction, or bio-learning-causal-inference-and-mendelian-randomization when the user wants maintained learning content organized by concrete AI, modeling, or causal-inference function.
- Use scientific-computing-router instead when the problem is mainly plotting, simulation, or numerical methods rather than ML.

Examples:
- choose the right ML or tabular analysis skill
- help me evaluate this model
- I need a forecasting or feature-importance workflow
