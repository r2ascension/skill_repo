# software-engineering-router

Use this router when:
- the task is general coding or application engineering work
- the request mentions frameworks, testing, debugging, performance, CI, or architecture without already naming a precise stack
- the main challenge is implementation quality, automation, or maintainability even if the surrounding domain is scientific
- you need to route among frontend, backend, language, and tooling skills

Routing guidance:
- Use framework and language skills such as django-patterns, laravel-patterns, nestjs-patterns, python-patterns, or rust-patterns once the stack is known.
- Use testing and verification skills such as python-testing, rust-testing, tdd-workflow, or verification loops for QA-heavy work.
- Use frontend-patterns, frontend-ui-engineering, or frontend-design-direction for UI work.
- Use MCP and infrastructure skills such as mcp-server-patterns, ci-cd-and-automation, observability-and-instrumentation, or security-and-hardening for systems and platform work.
- Use bioinformatics-router alongside this one when the code is an omics or assay pipeline and the biological analysis choice matters as much as the implementation.

Examples:
- help me debug this Django app
- which coding skill should handle this Rust refactor
- I need the right testing or framework skill for this codebase
